# Plán: doplnění parity SDK podle `docs/`

## Kontext
- Prošel jsem celý adresář `docs/` a porovnal ho s aktuálním SDK v `lib/src/repositories`, veřejnými exporty a několika klíčovými modely.
- Repo je Flutter/Dart balíček s centrálním `FakturoidClient` a jednotlivými repository třídami.
- Aktuální testy jsou jen výchozí placeholder v `test/fakturoid_api_test.dart`, takže bude potřeba doplnit reálné pokrytí SDK.

## Největší zjištěné mezery vůči dokumentaci
- `lib/src/repositories/number_formats_repository.dart`: používá `/number_formats.json`, ale docs uvádí `/number_formats/invoices.json`.
- `lib/src/repositories/webhooks_repository.dart`: endpoint pro failed deliveries má jinou cestu než docs (`/webhooks/{uuid}/failed_deliveries.json`), navíc vrací `dynamic` místo typovaného modelu.
- `lib/src/repositories/inbox_files_repository.dart`: create endpoint neumí předat `send_to_ocr`, i když docs to podporují už při uploadu.
- `lib/src/repositories/invoice_messages_repository.dart`: chybí podpora `deliver_now`.
- `lib/src/repositories/invoices_repository.dart`: search neumí `tags`; filtrování `document_type` nejspíš neposílá dokumentované hodnoty pro index (`regular|proforma|correction|tax_document`).
- `lib/src/repositories/expenses_repository.dart`: search neumí `tags`.
- `lib/src/repositories/inventory_moves_repository.dart`: globální index nepokrývá všechny dokumentované filtry (`updated_since`, `updated_until`, `inventory_item_id`); item-scoped list endpoint nevychází z docs úplně jistě.
- `lib/src/models/inventory_move.dart`: chybí read-only `document` objekt popsaný v docs.
- `lib/fakturoid_api.dart`: zatím neexportuje nic pro typovaný webhook failed-deliveries model.
- `test/fakturoid_api_test.dart`: placeholder test neověřuje nic relevantního.

## Doporučený postup implementace
1. Opravit jednoznačné endpoint/path nesoulady v repository vrstvách:
   - `lib/src/repositories/number_formats_repository.dart`
   - `lib/src/repositories/webhooks_repository.dart`
2. Doplnit chybějící request možnosti podle docs:
   - `lib/src/repositories/inbox_files_repository.dart` (`send_to_ocr` při create)
   - `lib/src/repositories/invoice_messages_repository.dart` (`deliver_now`)
   - `lib/src/repositories/invoices_repository.dart` (`tags`, správné mapování invoice index `document_type` filtru)
   - `lib/src/repositories/expenses_repository.dart` (`tags`)
   - `lib/src/repositories/inventory_moves_repository.dart` (`updated_since`, `updated_until`, `inventory_item_id` a ověření item-scoped listu)
3. Doplnit chybějící modelovou vrstvu pro nově podporované payloady:
   - typované modely pro webhook failed deliveries + export v `lib/fakturoid_api.dart`
   - rozšíření `lib/src/models/inventory_move.dart` o read-only `document`
4. Projít `lib/src/models/invoice.dart` a `lib/src/models/expense.dart` a rozhodnout, které novější atributy z docs/changelogu mají být součástí stejné implementační vlny, aby repository změny nevracely neúplně typovaná data.
5. Nahradit placeholder testy smysluplnými unit testy pro serializaci a skládání requestů v `test/fakturoid_api_test.dart` nebo v nových test souborech.
